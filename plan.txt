Right now the framework assumes that you need to either cache a whole node or not cache it at all. And I was having problems trying to figure out how to fall back from dirty slice instructions to full slice instructions in the case where an output is not cached. But I don't need to do that. I can just compile the dirty graph as a stand-alone graph that doesn't need to fall back if I work backwards using inputSlices starting at the root. So (1) forward pass to get dirty region of root node (2) backward pass using input slices to find the "needed" regions (3) plan based on "needed" regions and then you won't need to fall back.

1. forward pass all the way through to get dirty region for root node
2. backward pass to determine needed regions
needed[rootId] = root dirty region
updateNeeded(nodeId,Graph graph, ShapeProp prop,<uint32_t, std::vector<Region>> needed)
  node = graph.nodes[nodeId]
  parentNeeded = prop.backward(needed[nodeId])
  for i in range(node.parentIds):
    currentNeeded = needed.find or []
    needed[node.parentIds[i]] = mergeRegions(currentNeeded, parentNeeded[i]) // iterative per-dimension interval coalescing (group-by-other-dims + merge-1D)
    updateNeeded(node.parentIds[i],graph,prop,needed)

and then for cache, we just optimize "which combination of nodes being in cache is most valuable given the # of times each bucket has been run in historical logs". and then because the combination of cached nodes is static, we can compile buckets to skip unneeded instructions. Execution will just be a simple loop over memory allocation & running kernels. 

Problem: Stuff like sum will expand needed regions.
Imagine add -> pow -> mul -> sum -> add. updateNeeded, everything before sum will be full. But if we have cache, then we can get away with just computing slices of add,pow,mul before sum (this is what kv cache does I think). So we need to know what is cached at compile time so we know which operations we can replace with just computing the dirty slice instead of the full needed slice.

planning is different for different cache combinations because if a node is cached and the dirty region is smaller than the needed region, we can compute a slice smaller than the needed region (see add,pow,mul in example above). Basically, if a node is cached, we should update the EGraph with slice->contiguous->operation->scatter. can also add rewrite rule that scatter->slice can be cut out so we can do (slice->contiguous->operation->scatter->slice->contiguous->operation->scatter) = (slice->contiguous->operation->operation->scatter).
Egraph-extraction should be aware of how much memory is remaining (total - sum of cached nodes). It should prune any options that go over this memory limit. If it cannot find any options, it should throw a custom error and we will skip that combination.

Optimizing search space of cached/uncached combinations:
-max mem check: for each device (sum of memory used by combination + peak memory used by plan < max_memory_bytes)
-find lower/upper bound on # of cached nodes. this transforms it O(2^N) -> O(2^(N_UPPER-N_LOWER)).
--to find upper bound: sort nodes by mem size, accumulate from smallest to largest until adding the next one would put you over the memory limit where the memory limit is sum(memory limit for each device)
--to find lower bound: call plan with no cached nodes, see peak memory it requires, then same as to find upper limit but subtract the memory limit by peak mem

For memory management during execution, we won't need the complex tryEvict logic anymore. Cached nodes will be locked, any other transient nodes will be locked according to refCount. If we can't find a slot, defragment memory. If we still can't find a spot after defrag, then throw error, but that won't happen because the extraction is memory-aware.

to find peak mem:
```
current_mem = sum(cached nodes)
peak_mem = 0

for node in topo_order:
    // allocate output
    if node not in cached nodes:
      current_mem += size[node]

    // parents might die after this op
    for p in parents[node]:
        uses[p]--
        if uses[p] == 0 and p not in cached nodes:
            current_mem -= size[p]

    peak_mem = max(peak_mem, current_mem)
```
